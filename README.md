# Yandex Black Friday — pymongo-api

Проект демонстрирует приложение pymongo-api с шардированным кластером MongoDB, репликацией и кешированием Redis.

Финальная реализация находится в директории `sharding-repl-cache/`.

## Архитектура

### Схема инфраструктуры (CDN)

> Для просмотра схемы откройте файл [schemes/cdn.drawio](schemes/cdn.drawio) в [draw.io](https://app.diagrams.net/) или в расширении Draw.io для VS Code.

### Компоненты

| Компонент                              | Описание                         | Порт  |
| -------------------------------------- | -------------------------------- | ----- |
| **configSrv1, configSrv2, configSrv3** | Реплика-сет сервера конфигурации | 27017 |
| **shard1-1, shard1-2, shard1-3**       | Реплика-сет первого шарда        | 27018 |
| **shard2-1, shard2-2, shard2-3**       | Реплика-сет второго шарда        | 27019 |
| **mongos_router**                      | Роутер MongoDB                   | 27020 |
| **redis**                              | Кеш-сервер Redis                 | 6379  |
| **pymongo_api**                        | Приложение                       | 8080  |

## Как запустить стенд

### 1. Перейти в директорию финальной реализации

```shell
cd sharding-repl-cache
```

### 2. Запустить контейнеры

```shell
docker compose up -d
```

Дождитесь, пока все контейнеры успешно стартуют (15-20 секунд):

```shell
docker compose ps
```

### 3. Инициализация кластера и наполнение данными

```shell
./scripts/mongo-init.sh
```

Скрипт выполняет:

1. Инициализирует реплика-сет сервера конфигурации (`config_server`) из 3 нод.
2. Инициализирует реплика-сет первого шарда (`shard1`) из 3 нод.
3. Инициализирует реплика-сет второго шарда (`shard2`) из 3 нод.
4. Подключается к роутеру и:
   - добавляет оба шарда в кластер;
   - включает шардирование для базы `somedb`;
   - шардирует коллекцию `helloDoc` с хешированным ключом `name`;
   - вставляет 1000 тестовых документов.
5. Выводит количество документов на каждом шарде.
6. Выводит статус реплика-сетов (primary/secondary для каждой ноды).
7. Проверяет кеширование: выполняет 3 запроса к `/helloDoc/users` и замеряет время.
